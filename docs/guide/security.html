Security

LiteSkill VR handles sensitive vulnerability-research data. The threat model below describes what the software defends against — and what it explicitly leaves to the operating system.

No at-rest encryption is provided by the software itself. This must be done at the OS level (LUKS, FileVault, BitLocker, etc.).

Threat model

ThreatMitigation
Malicious agent outputAgent-created content is rendered as sanitized markdown. No raw HTML execution.
MCP server exposureBound to 127.0.0.1 only. Not accessible from the network.
Malicious graph importsImport parsers validate and sanitize input. No code execution from imported data.
Supply chain (dependencies)Tauri's Rust backend minimizes JS dependency surface. CSP enforced in the webview.

Data handling

  • All project data stored locally in SQLite — no cloud sync.
  • Export files may contain sensitive vulnerability details — the user is warned before export.
  • All MCP mutations are logged with author, author_type, and timestamp.
  • bulk_delete requires at least one filter to prevent an accidental full wipe.

Content Security Policy

The Tauri webview enforces a strict CSP: no inline scripts, no eval, no external resource loading (fonts and images are loaded from the app bundle), and connect-src limited to localhost (the MCP server).
