File Formats

Implementation status. Only the .lsvr project file is implemented. Every export and import format below is planned — not yet built (there is no export/import code in the repo today), and OS file-association is not wired up either.

Project files

Each project is a single .lsvr file (a SQLite database — WAL mode, foreign keys on). Projects can be backed up, shared, or archived by copying the file.

Status. The desktop app currently opens (or creates) project.lsvr in its working directory. Planned: an open/new file dialog and registering the .lsvr extension with the OS so double-clicking opens the project.

Export formats — planned

None of the export formats below are implemented yet.

Markdown report

A human-readable report generated from project data — items, items of interest (with severity and tags), notes, and connections, grouped per item.

# [Project Name] — Research Report

## Item: httpd (ELF, arm32)

**Status**: reviewed
**Path**: /usr/bin/httpd

### Items of Interest

#### parse_header() [Critical] [memory-corruption]
**Location**: 0x08041234

No bounds check on Content-Length header...

### Connections
- httpd → libfoo.so [links]: dynamically links libfoo.so

JSON export

A machine-readable full project export conforming to the data model.

{
  "format": "liteskill-vr",
  "version": "1.0",
  "exported_at": "2026-04-13T00:00:00Z",
  "project": { ... },
  "tags": [ ... ],
  "connection_types": [ ... ],
  "items": [ ... ],
  "notes": [ ... ],
  "items_of_interest": [ ... ],
  "connections": [ ... ]
}

SARIF

Static Analysis Results Interchange Format — for integration with GitHub Security, Azure DevOps, and other tools. Each item of interest maps to a SARIF result, severity to level, and tags to taxa.

Import formats — planned

Not implemented yet.

  • SARIF import — import results from static-analysis tools as draft items of interest for review.
  • LiteSkill VR JSON — re-import previously exported projects for merging or archiving.
