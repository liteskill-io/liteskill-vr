File Formats

Project files

Each project is a single .lsvr file (a SQLite database). The .lsvr extension is registered with the OS so double-clicking opens the project in LiteSkill VR.

Export formats

Markdown report

A human-readable report generated from project data — items, items of interest (with severity and tags), notes, and connections, grouped per item.

# [Project Name] — Research Report

        ## Item: httpd (ELF, arm32)

        **Status**: reviewed
        **Path**: /usr/bin/httpd

        ### Items of Interest

        #### parse_header() [Critical] [memory-corruption]
        **Location**: 0x08041234

        No bounds check on Content-Length header...

        ### Connections
        - httpd → libfoo.so [links]: dynamically links libfoo.so

JSON export

A machine-readable full project export conforming to the data model.

{
          "format": "liteskill-vr",
          "version": "1.0",
          "exported_at": "2026-04-13T00:00:00Z",
          "project": { ... },
          "tags": [ ... ],
          "connection_types": [ ... ],
          "items": [ ... ],
          "notes": [ ... ],
          "items_of_interest": [ ... ],
          "connections": [ ... ]
        }

SARIF

Static Analysis Results Interchange Format — for integration with GitHub Security, Azure DevOps, and other tools. Each item of interest maps to a SARIF result, severity to level, and tags to taxa.

Import formats

  • SARIF import — import results from static-analysis tools as draft items of interest for review.
  • LiteSkill VR JSON — re-import previously exported projects for merging or archiving.
