Headless MCP Server

A second binary, liteskillvr-mcp, runs just the MCP interface against a .lsvr project file — without the GUI or any Tauri system dependencies. It supports both HTTP and stdio transports.

Pre-built binaries for Linux, macOS (x86_64 and aarch64), and Windows ship with each release as liteskillvr-mcp-headless-<target-triple>[.exe]. The Linux .deb / .rpm packages also install it to /usr/bin/liteskillvr-mcp alongside the desktop app.

Running it

# HTTP transport (default) — agents connect to http://127.0.0.1:<port>/mcp
liteskillvr-mcp path/to/project.lsvr
liteskillvr-mcp --port 27182 path/to/project.lsvr

# stdio transport — agents spawn the binary and talk JSON-RPC over stdin/stdout
liteskillvr-mcp --stdio path/to/project.lsvr

# Create the project file if it doesn't exist
liteskillvr-mcp --init path/to/project.lsvr

Build from source

# No GUI deps needed
cd src-tauri
cargo build --release --bin liteskillvr-mcp --no-default-features
./target/release/liteskillvr-mcp path/to/project.lsvr

Building with --no-default-features drops the gui feature (which gates Tauri), producing a truly headless binary with no WebKitGTK/Tauri link-time dependencies.

Claude Code configuration

HTTP transport:

{
  "mcpServers": {
    "liteskill": {
      "url": "http://127.0.0.1:27182"
    }
  }
}

stdio transport:

{
  "mcpServers": {
    "liteskill": {
      "command": "liteskillvr-mcp",
      "args": ["--stdio", "/path/to/project.lsvr"]
    }
  }
}

The headless binary uses the same tool surface as the embedded server — the MCP tools reference applies identically. In stdio mode the author header isn't available, so all writes are attributed to stdio-agent.
