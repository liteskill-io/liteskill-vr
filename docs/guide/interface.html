The Interface

The UI is built for rapid navigation across a large project: hop between items, search for patterns, review AI-generated findings, and correct mistakes without losing context.

Design philosophy

  • Tabs, not panels — each item opens in a tab, like a browser.
  • Speed over structure — every view reachable in 1–2 keystrokes.
  • Search everything — full-text search across notes, items of interest, and connections.
  • Minimal chrome — maximize content area; collapse the rest on demand.

Layout

┌──────────────────────────────────────────────────────────┐
        │ ◄ ►  Breadcrumb: project > httpd > parse_header           │
        ├──────────────────────────────────────────────────────────┤
        │                  Active Tab (full bleed)                   │
        │   Item Detail / Connection Map / Search Results           │
        ├──────────────────────────────────────────────────────────┤
        │  httpd  ▸ libfoo.so  ▸ httpd.conf  ▸ init.sh  [+]         │
        ├──────────────────────────────────────────────────────────┤
        │  MCP: listening  │  12 items  │  3 critical  │  5 high     │
        └──────────────────────────────────────────────────────────┘

Views

Project overview

The home view when no item tab is focused: a list/grid of all items, grouped by item_type or flat, color-coded by analysis status, with quick stats per item and a tag-filter sidebar.

Item detail

The main view when an item tab is active, in three sections:

  • Header — name, type, path, architecture, status, tags; editable inline, with a delete button.
  • Items of interest — each shows title, severity badge, location, description preview, and tags; click to expand, inline edit/delete, n to add.
  • Notes — markdown-rendered, with author (human/agent) badge, timestamp, and tags; a to add.
  • Connections — source → target with type and description; click to navigate to the other end.

Connection map

A project-wide Cytoscape.js graph: items are nodes, connections are edges, edge labels show the connection type, nodes are colored by analysis status and sized by IOI count. Click a node to open it as a tab. Useful for seeing how components relate across a firmware image.

Tag manager & search

The tag manager lists every registered tag with description, color, and usage count, and lets you create, edit, and delete them. Search results are grouped by entity type with highlighted snippets and filters for entity type, severity, and tags.

Keyboard shortcuts

KeyAction
⌘K / Ctrl+KCommand palette (search everything)
nNew item of interest on current item
aNew note on current item
cNew connection from current context
Tab / Shift+TabCycle between open tabs
Ctrl+[ / Ctrl+]History back / forward
EscDismiss overlay / go up one level
?Show the keyboard-shortcut cheatsheet
Delete / BackspaceDelete selected entity (with confirmation)

When an agent creates, updates, or deletes entities via MCP, the UI updates immediately via Tauri IPC events. Dark mode is the default; light mode is available, and code snippets in notes are syntax-highlighted with Shiki.
