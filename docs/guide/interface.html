The Interface

The UI is built for rapid navigation across a large project: hop between items, search for patterns, review AI-generated findings, and correct mistakes without losing context.

Core requirement: human/agent parity. There must be zero things an AI agent can do that a human cannot do in the UI. Every mutating MCP tool has a human CRUD affordance (human ≥ agent), enforced in CI (task parity:check). The UI is a full read-write client: reads come from project_snapshot; writes go through a single mcp_call(tool, args) command that runs the same dispatch as the MCP server, stamped author_type: "human", refreshing on db-changed. Still planned: the command palette, markdown rendering, tab badges/reordering, and breadcrumb/history.

Design philosophy

  • Tabs, not panels — each item opens in a tab, like a browser.
  • Speed over structure — every view reachable in 1–2 keystrokes.
  • Search everything — full-text search across notes, items of interest, and connections.
  • Minimal chrome — maximize content area; collapse the rest on demand.

Layout

Views

Project overview

The home view when no item tab is focused: a list/grid of all items, grouped by item_type or flat, color-coded by analysis status, with quick stats per item and a tag-filter sidebar.

Item detail

The main view when an item tab is active — read-only today; the inline edit/add/delete affordances below are planned. Three sections:

  • Header — name, type, path, architecture, status, tags (planned: inline edit + delete).
  • Items of interest — each shows title, severity badge, location, description preview, and tags; click to expand (planned: inline edit/delete, n to add).
  • Notes — author (human/agent) badge, timestamp, and tags; shown as plain text today (planned: markdown rendering, a to add).
  • Connections — source → target with type and description; click to navigate to the other end.

Connection map

A project-wide Cytoscape.js graph: items are nodes, connections are edges, edge labels show the connection type, nodes are colored by analysis status and sized by IOI count. Click a node to open it as a tab. Useful for seeing how components relate across a firmware image.

Tag manager & search — planned

Neither is in the UI yet. Tags are shown read-only on items and findings; managing tags and running full-text search / structured filter happen over MCP today (see MCP tools). The intended UI: a tag manager to create/edit/delete tags, and a grouped search-results view with highlighted snippets.

Keyboard shortcuts

Planned, except zoom. Only Ctrl/⌘ +/-/0 (zoom) are wired up today; the rest of this table is a design target.

KeyAction
⌘K / Ctrl+KCommand palette (search everything)
nNew item of interest on current item
aNew note on current item
cNew connection from current context
Tab / Shift+TabCycle between open tabs
Ctrl+[ / Ctrl+]History back / forward
EscDismiss overlay / go up one level
?Show the keyboard-shortcut cheatsheet
Delete / BackspaceDelete selected entity (with confirmation)

When an agent creates, updates, or deletes entities via MCP, the UI updates immediately (it re-fetches the snapshot on the db-changed event). Dark mode is the only theme today; a light mode, markdown rendering of notes, and syntax highlighting are planned.
