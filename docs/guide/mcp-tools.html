MCP Tools

LiteSkill VR hosts an MCP (Model Context Protocol) server that starts automatically when the app opens. It is a read-write interface: agents don't just dump findings — they query existing research, review what past sessions discovered, build on prior analysis, and avoid duplicating work.

For connecting an agent and running a session, see Working with AI agents; this page is the tool reference.

Connection

The server runs on HTTP (127.0.0.1) using the MCP streamable-HTTP transport at the /mcp endpoint, and the port is shown in the status bar. Configure Claude Code:

{
          "mcpServers": {
            "liteskill": {
              "url": "http://127.0.0.1:27182"
            }
          }
        }

Each agent identifies itself with X-LiteSkill-Author: claude-code on every request; entities it creates are stamped with that author and author_type: "agent". See also the headless binary, which exposes the identical tool surface.

Design principles

  1. Read before write — project_summary and filter orient the agent before it creates anything.
  2. Batch writes — all creates have all-or-nothing batch variants.
  3. Duplicate detection — ioi_create warns on similar entries.
  4. Registered vocabularies — tags and connection types must exist first.
  5. Two query modes — search (full-text) and filter (structured).
  6. Session continuity — changes_since catches an agent up.
  7. Mistake recovery — bulk_delete undoes a bad session.

Typical agent session

1. Orient
           project_summary()      → current state of all research
           tag_list()             → available tags
           connection_type_list() → available connection types

        2. Catch up (if not first session)
           changes_since("2024-01-15T...")

        3. Review a target
           item_get(id)           → everything known about this item

        4. Analyze (via external tools, e.g. pyghidra-mcp)

        5. Document findings (batch)
           ioi_create_batch(item_id, [...])
           note_create(item_id, ...)

        6. Draw connections (batch)
           connection_create_batch([...])

        7. Structured queries
           filter(entity_type="item_of_interest", severity="critical")

        8. Correct mistakes
           bulk_delete(author="claude-code", since="...", entity_type="item_of_interest")

Batch errors

If any entry in a batch fails validation, the whole batch is rejected:

{
          "error": "batch_validation_failed",
          "index": 3,
          "message": "Tag 'buffer-overflow' is not registered. Did you mean 'memory-corruption'?",
          "suggestion": "Call tag_list() to see registered tags, or tag_create() to register a new one."
        }
